# Handoff plan template

Plans target an executor with no session context. They must be self-contained, bounded, and verifiable.

Name files `plans/NNN-short-slug.md` in execution order.

- [Plan template](#plan-template)
- [Index template](#index-template)
- [Quality check](#quality-check)


## Plan template

```markdown
# Plan NNN: <imperative title>

> Follow this plan step by step. Run each verification before moving on. If a STOP condition occurs, stop and report. Do not improvise.
>
> Drift check:
> - committed changes: `git diff --stat <planned-at SHA>..HEAD -- <in-scope paths>`
> - uncommitted changes: `git diff --stat -- <in-scope paths>`
> If either command shows in-scope changes and Current State no longer matches, STOP.

## Status

- Priority: P1 | P2 | P3
- Effort: S | M | L
- Risk: LOW | MED | HIGH
- Depends on: `plans/NNN-*.md` or none
- Category: bug | security | perf | tests | tech-debt | migration | dx | docs | direction
- Planned at: commit `<short SHA>`, <YYYY-MM-DD>
- Issue: <URL; only for `--issues`>

## Why this matters

2-5 sentences: problem, concrete cost, expected improvement.

## Current state

- Relevant files, roles, and short excerpts with `file:line` (no prior references).
- Repo conventions to match, plus ADR/product/design constraints.

## Commands

| Purpose | Command | Expected |
|---|---|---|
| Install | `<repo command>` | exit 0 |
| Typecheck | `<repo command>` | exit 0 |
| Test | `<repo command>` | relevant tests pass |
| Lint | `<repo command>` | exit 0 |

Use commands verified during recon.

## Scope

In scope:

- `<file>`

Out of scope:

- `<file or area>` - why.

## Git workflow

- Branch: `<repo convention or advisor/NNN-slug>`
- Commit style: `<repo convention and example>`
- Do not push or open a PR unless instructed.

## Steps

### Step 1: <imperative title>

Exact files, symbols, target shape, and constraints.

Verify: `<command>` -> `<expected result>`

### Step 2: ...

## Test plan

- Tests to add/change, with file paths and cases.
- Existing test to model, if any.
- Verification command and expected result.

## Done criteria

- [ ] `<typecheck command>` exits 0
- [ ] `<test command>` exits 0 and covers `<case>`
- [ ] No files outside Scope changed

## STOP conditions

- Current State does not match live code.
- Verification fails twice after reasonable fixes.
- Fix requires out-of-scope files.
- Key assumption `<assumption>` is false.

## Maintenance notes

- Future implications.
- Review focus.
- Deferred follow-ups and why.
```

## Index template (plans/README.md)

```markdown
# Implementation Plans

Generated by the consilium skill on <date>. Execute in order unless dependencies say otherwise.

| Plan | Title | Priority | Effort | Depends on | Status |
|---|---|---|---|---|---|
| 001 | ... | P1 | S | - | TODO |

Status: TODO | IN PROGRESS | DONE | BLOCKED (<reason>) | REJECTED (<reason>)

## Dependency notes

- `<plan>` requires `<plan>` because `<reason>`.

## Findings Considered And Rejected

- `<finding>`: not worth doing because `<reason>`.
```

## Quality check

Ensure: executable by fresh model; command+expected result for every verification; exact files/symbols named; specific STOP conditions; no secrets; drift check matches scope.
